Detailed Description

FirebaseAuth provides methods for authenticating on the Firebase and also managing the accounts such as changing password or email, sending password reset requests and updating profile data. Authentication is necessary if the rules on the Firebase project are set in such a way that authentication is required for writing to the database. After logging in, the property currentUser holds a reference to the authenticated user of type FirebaseUser.

Property Documentation

apiKey : string

This property holds the API Key of the Firebase project, passed from FirebaseApp.


currentUser : FirebaseUser

Holds a reference to the user that is currently logged in. Typically used to access the idToken of the user for actions that require authentication, for example:

FirebaseApp { id: fbApp }
FirebaseAuth {
    id: fbAuth
    apiKey: fbApp.apiKey

    onSignedIn: {
        fbAuth.getUserData(currentUser.idToken)

        // Fetches audio data from database
        fbDb.getValue("/Audio/.json", fbAuth.currentUser.idToken, codes.audioDataCode)

        // Adds the listeners to these locations of the database
        fbDb.listenEvents("/Audio/.json", fbAuth.currentUser.idToken, codes.audioDataCode)
        fbDb.listenEvents("/Users/.json", fbAuth.currentUser.idToken, codes.userDataCode)
    }
}

FirebaseDatabase {
    id: fbDb
    ...
}

See also FirebaseUser.


Signal Documentation

Emitted when an error ocurred with the corresponding error in human readable format.

Note: The corresponding handler is onErrorOcurred.


signedIn()

Emitted when a user successfully authenticates.

Note: The corresponding handler is onSignedIn.

See also currentUser.


Method Documentation

void changeEmail(string idToken, string newEmail)

Changes the email of the user with idToken to the new email newEmail.

See also changePassword().


void changePassword(string idToken, string newPassword)

Changes the password of the user with idToken to the new password newPassword.

See also changeEmail().


void deleteAccount(string idToken)

Permanently deletes the profile with idToken from the Firebase.


void exchangeRefreshToken(string refreshToken)

Exchanges a refreshToken for an ID token. When the user authenticates, both of these tokens are received. The ID token is necessary for all requests to the Firebase but has a short lifetime of 1 hour and needs to be refreshed with the refresh token by calling this method. When the response is received, the ID token is automatically updated on the currentUser.


void getUserData(string idToken)

Retrieves information about the user with idToken, tipically the currentUser and automatically updates its properties.


void profileUpdate(string idToken, string name, string photoUrl)

Updates the fields name and photoUrl of the user with idToken.


void sendEmailVerification(string idToken)

Sends verification email to the user with idToken.

See also sendPasswordResetEmail().


void sendPasswordResetEmail(string email)

Sends request for password reset to email.


void signInWithEmailAndPassword(string email, string password)

Calls a login request with email and password. If successful emits signals signedIn() and updates currentUser to the user, otherwise if an error occurs (e.g incorrect email/password, no network access) the signal errorOcurred is emited with the error string.


void signInWithOAuthCredential(string authToken, string providerId)

Authenticates user with Google's authToken and providerId.


void signUpWithEmailAndPassword(string email, string password, string name)

Registers a new account in the Firebase of user with name, email and password. If successful emits signals signedIn() otherwise if an error occurs (e.g invalid email/password, no network access) then the signal errorOcurred() is emited with the error string.

Note: password must be a minimum of 6 characters.
