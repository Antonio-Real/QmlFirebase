Detailed Description

FirebaseDatabase provides methods for adding, updating and deleting data from the Firebase Realtime Database. Moreover, it also provides the method listenEvents that listens to events in a given database path and fires a signal when an event occurs.

Property Documentation

apiKey : string

This property holds the API Key of the Firebase project, obtained from FirebaseApp.


databaseUrl : string

This property holds the database where the Firebase Realtime Database resides, obtained from FirebaseApp.


Signal Documentation

dataEvent(string data, int requestCode)

Emitted when an event from a listener occurs, sends the server response data and the corresponding requestCode.

Note: The corresponding handler is onDataEvent.

See also listenEvents().

Method Documentation

void deleteValue(string dbPath, string idToken)

Deletes all data in path dbPath with idToken if the Firebase Database rules require authentication.


void getValue(string dbPath, string idToken, int requestCode)

Requests all the data in path dbPath with idToken if the Firebase Database rules require authentication.

When the request is successful the signal dataRetrieved() is triggered with the contents of the server response atached to the code requestCode. The request code provides a way to differentiate where the response comes from, as seen in the example:

FirebaseDatabase {
    id: fbDb
    ...
    onDataRetrieved: { // params (data, requestCode)
        if(requestCode == 10)
            console.log(data) // data contains event from Users path
        else if(requestCode == 20)
            console.log(data) // data contains event from Alerts path
    }
    Component.onCompleted: {
        fbDb.getValue("/Users/.json", "insert-if-necessary", 10)
        fbDb.getValue("/Alerts/.json", "insert-if-necessary", 20)
    }
}

See also dataRetrieved().


void listenEvents(string dbPath, string idToken, int requestCode, bool ignoreFirstEvent, bool recursive)

Registers a listener to the database path dbPath with idToken if the Firebase Database rules require authentication.

When an event occurs (e.g data was added, updated or deleted) on that path, the signal dataEvent() is triggered with the contents of the server response atached to the code requestCode. The request code provides a way to differentiate where the response comes from, as seen in the example:

FirebaseDatabase {
    id: fbDb
    ...
    onDataEvent: { // params (data, requestCode)
        if(requestCode == 10)
            console.log(data) // data contains event from Users path
        else if(requestCode == 20)
            console.log(data) // data contains event from Alerts path
    }
    Component.onCompleted: {
        fbDb.listenEvents("/Users/.json", "insert-if-necessary", 10)
        fbDb.listenEvents("/Alerts/.json", "insert-if-necessary", 20)
    }
}

When an event listener is first registered, the first event will always return the entire contents of the chosen path. To ignore the first event, set ignoreFirstEvent to true (if no parameter passed, is true by default).

If you want the listener to be recursive (i.e it re-registers itself when the server closes connection) set the argument recursive to true.

See also dataEvent().


void pushValueWithUniqueKey(string dbPath, string jsonData, string idToken)

Writes to the database path dbPath a new entry with a randomized unique key and value jsonData.

FirebaseDatabase {
    id: fbDb
    ...
}

Button {
    text: "Push value"
    onClicked: fbDb.pushValueWithUniqueKey("/random.json", JSON.stringify({"text":"Foo Bar"}), fbAuth.currentUser.idToken)
}

In the example above, if the button is repeatedly clicked, multiple entries with random keys will be added to the path.

See also writeValue(), updateValue(), and deleteValue().


void updateValue(string dbPath, string jsonData, string idToken)

Updates the entry in dbPath with the new value in jsonData. The contents of jsonData can be used to update the entire entry or just patch one of the fields, for example:

FirebaseDatabase {
    id: fbDb
    ...
}

Button {
    text: "Push value"
    onClicked: fbDb.pushValueWithUniqueKey("/random/myData.json", JSON.stringify({"text":"Foo Bar", "myInt" : 10}), fbAuth.currentUser.idToken)
}
Button {
    text: "Update value"
    onClicked: fbDb.pushValueWithUniqueKey("/random/myData.json", JSON.stringify({"text":"Foo Bar"}), fbAuth.currentUser.idToken)
}

In the example above, if the value is pushed and then updated, only the text will be updated and "myInt" will remain the same.

See also writeValue().


void writeValue(string dbPath, string jsonData, string idToken)

Writes to the database path dbPath a new entry with a randomized unique key and value jsonData.

Warning: Careful use of this method is required since the last specifier of the path denotes the key of the new entry and all data in that path will be overwritten, for example:

FirebaseDatabase {
    id: fbDb
    ...
}
Button {
    id: button1
    onClicked: fbDb.writeValue("/random/myText.json", JSON.stringify({"text":"Foo Bar"}), fbAuth.currentUser.idToken)
}
Button {
    id: button2
    onClicked: fbDb.writeValue("/random.json", JSON.stringify({"text":"Foo Bar"}), fbAuth.currentUser.idToken)
}

In the example above, if button1 is pressed, a new entry with key myText and the given value, as expected. If button2 is pressed, the entire contents of the path "/random.json" will be replaced by the given value, as opposed to inserting the new entry in the "/random.json" path.

See also pushValueWithUniqueKey(), updateValue(), and deleteValue().
